3. Installation

Three ways, ranked by least friction. All of them end with the dashboard running at http://localhost:3001 and auto-starting at login. Nothing is uploaded — the whole install is local.

Option A — One-line Terminal install (recommended)

Open Terminal (⌘-Space, type "Terminal") and paste:

curl -L https://github.com/anorby515/AI2FI/archive/refs/heads/release.tar.gz | tar xz && cd AI2FI-release && bash dashboard/setup.command

Downloads, unpacks, installs. No Gatekeeper prompt, no rename step. The script runs Homebrew and Node checks, installs dependencies, builds the client, registers the login agent, and opens the dashboard in your browser. First run takes a couple of minutes.

Option B — Git clone

git clone --branch release https://github.com/anorby515/AI2FI.git
cd AI2FI
bash dashboard/setup.command

Tracks the release branch so git pull picks up new releases. Files cloned via git aren't quarantined, so double-clicking setup.command from Finder also works with no Gatekeeper dance.

Option C — ZIP download + Finder

Download AI2FI.zip

Unzip, then in Finder open AI2FI-release/dashboard/ and double-click setup.command.

Finder path hits Gatekeeper. macOS 15+ blocks .command files downloaded from the internet. You'll see a "cannot be opened" dialog — click Done, open System Settings → Privacy & Security, scroll to the Security section, click Open Anyway, and enter your password. Option A avoids this entirely.

After install

When the installer finishes, your browser opens http://localhost:3001 and you're looking at the dashboard. It auto-starts every time you log into your Mac.

The dashboard reads a local spreadsheet to render portfolio, net worth, and benchmark views. Put yours at:

user-profiles/<your-name>/private/Finances.xlsx

Start from user-profiles/example/ as a template if you want the expected shape. Everything under user-profiles/<your-name>/ is gitignored — it never leaves your machine.

To uninstall

bash dashboard/uninstall.command

Unloads the login agent, removes the plist, kills anything on port 3001. Your files, spreadsheet, and profile are untouched.

Non-Mac / run it manually

The installer uses launchd (macOS only), but the server itself runs anywhere Node does. You'll need Node 18+. From the repo root:

cd dashboard
npm install
cd client && npm install && npm run build && cd ..
npm start

Then open http://localhost:3001. Full details and the dev workflow are in dashboard/README.md.

4. Start coaching with Claude

The dashboard is the "seeing" half. Claude is the "thinking" half — and that's where the work happens. The entry point is a single skill.

Open Claude in this folder

Open Claude Code, Claude Desktop, or claude.ai/code with the AI2FI directory as the working folder. Claude automatically picks up the skills in skills/ and the curriculum in core/ and modules/.

Run the entry skill

In Claude, type:

/financial-check-in

That's the front door. It runs the master assessment, consults the financial order-of-operations, and routes you into whichever module matches where you are today. First-time users get the full two-pass assessment; returning users get the lightweight check-in.

Other skills you can call directly

  • /module-coach <module-name> — dives into one topic (e.g. /module-coach investing).
  • /knowledge-check <topic> <level> — standalone quiz, tiered fundamentals → intermediate → advanced.
Your data stays yours. Personal financial files live under user-profiles/<you>/private/ and are gitignored. Session notes, goals, and competency levels are written by Claude at session close — also gitignored. Nothing under user-profiles/ is ever committed except the example/ template.
